## 一、GLM系列模型技术架构演进

### 1.1 GLM核心架构创新（原始架构）

**技术来源**: [GLM论文 - arXiv:2103.10360](https://arxiv.org/abs/2103.10360) (ACL 2022)

#### 核心创新点:
1. **自回归空格填充 (Autoregressive Blank Infilling)**
   - 统一自然语言理解(NLU)、条件生成、无条件生成三类任务
   - 支持对多个文本片段(spans)的任意顺序预测
   - 不同于BERT的掩码语言建模和GPT的标准自回归

2. **2D位置编码**
   - 同时表示token在原始序列中的位置
   - 表示预测片段的生成顺序
   - 实现位置信息的二维表示

3. **空白配置任务适配**
   - 通过改变空白数量、长度、预测顺序适配不同任务类型
   - NLU任务: 少量短空白
   - 条件生成: 单个长空白
   - 无条件生成: 覆盖全文的多个空白

**技术优势**: 单一预训练模型统一多种任务架构，在相同模型规模和数据下性能超过BERT、T5、GPT [1]

### 1.2 GLM-4系列架构演进

#### **GLM-4.5** (2025年8月发布)
**官方技术报告**: [arXiv:2508.06471](https://arxiv.org/abs/2508.06471)

| 技术参数 | GLM-4.5 | GLM-4.5-Air |
|---------|---------|-------------|
| **总参数量** | 355B | 106B |
| **激活参数量** | 32B | 12B |
| **训练数据量** | 23T tokens | 23T tokens |
| **上下文窗口** | 128K | 128K |
| **隐藏层维度** | 5120 | 4096 |
| **注意力头数** | 96 | 96 |
| **专家总数** | 160 | 128 |
| **每token激活专家数** | 8 | 8 |

**架构创新**:
- **MoE (混合专家)架构**: 采用sigmoid门控路由和无损平衡损失
- **深度优先设计**: 减少宽度(隐藏维度、路由专家数)同时增加层数 - 发现更深模型具有更好的推理能力
- **注意力机制**: 分组查询注意力(GQA) + 部分RoPE + QK-Norm
  - 96个注意力头(5120维度) - 是同类模型的2.5倍
  - QK-Norm稳定注意力logits
- **多Token预测(MTP)**: 添加MoE层作为MTP层支持推测解码
- **训练优化**: 使用Muon优化器(除embedding、bias、RMSNorm权重外所有参数)
  - Muon设置: Newton-Schulz步数=5, momentum=0.95, update RMS=0.2

**训练流程** [2]:
1. **预训练(4K上下文)**: 15T通用网络数据 + 7T代码/推理专项数据
2. **中间训练(扩展至128K)**:
   - 仓库级代码训练(拼接同仓库代码文件)
   - 合成推理数据(数学/科学/编程竞赛Q&A)
   - 长上下文与Agent训练(最长128K)
3. **后训练**: 冷启动SFT → 整体SFT(百万样本) → 强化学习

**参数效率**: 参数量仅为DeepSeek-R1的1/2、Kimi-K2的1/3，但性能相当或更优 [2]

#### **GLM-4.7** (2025年12月发布)

| 参数 | 规格 |
|------|------|
| **总参数量** | ~358B MoE |
| **激活参数量** | ~32B |
| **上下文窗口** | 200K |
| **最大输出Tokens** | 128K |
| **轻量版** | GLM-4.7-Flash (30B-A3B) |

**架构创新**: 引入三种思考模式 [4]:
1. **交错式思考(Interleaved Thinking)**: 每次回答/工具调用前都思考
2. **保留式思考(Preserved Thinking)**: 多轮对话中自动保留思考块，提升缓存命中率
3. **轮级思考(Turn-level Thinking)**: 按"轮"控制推理开销

**性能提升** (相比GLM-4.6) [4]:
- **编码**: SWE-bench 73.8% (+5.8%), SWE-bench多语言66.7% (+12.9%), Terminal Bench 2.0 41% (+16.5%)
- **推理**: HLE 42.8% (+12.4%), AIME 2025 95.7%
- **工具调用**: τ²-Bench 84.7%, BrowseComp 67分
- **前端**: PPT 16:9适配率从52%提升至91%

**性能表现** [5]:

### 1.3 GLM架构核心技术总结

**技术护城河**:
1. **原创GLM架构**: 自回归空格填充 + 2D位置编码，ACL 2022发表
2. **MoE优化能力**: loss-free balance路由 + sigmoid门控 + QK-Norm
3. **思考模式创新**: 交错/保留/轮级三种思考模式
4. **训练框架**: 开源强化学习框架"Slime"，支持异步智能体RL
5. **参数效率**: 用更少激活参数达到与更大模型相当性能
